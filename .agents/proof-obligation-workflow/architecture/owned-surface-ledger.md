| Behavior                                                        | Local owner | Delegated to                      | Why delegate or own                                                                                                                                     | Residual local obligations                                                                                                                                                                                                                 | Tests required                                                                                                                                         |
| --------------------------------------------------------------- | ----------- | --------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Markdown knowledge graph storage and link-aware note operations | false       | IWE CLI                           | The transcript selects IWE as the authoritative graph/refactoring layer to avoid building a database, parser, backlink engine, or MCP layer.            | Invoke IWE in the configured vault directory; fail if IWE is missing or the vault is not initialized.                                                                                                                                      | Real CLI test creates a vault and retrieves a note through the wrapper/IWE boundary.                                                                   |
| Search across project/global scopes                             | true        | IWE `find` plus `rg` subprocesses | IWE owns fuzzy title/key graph search; `rg` owns lexical body search through normal filesystem tooling.                                                 | Restrict IWE graph anchors and `rg` roots to exact configured scope directories; return concrete keys for title/key matches and path/line matches for body hits.                                                                           | Real CLI tests write project and global notes and prove `--scope` changes which body and title/key hits are returned.                                  |
| Vault layout creation                                           | true        | Filesystem plus IWE init          | The wrapper owns the machine-level convention: `global/`, `projects/`, `inbox/`, `templates/`, `_meta/`, and root indexes.                              | Create the complete layout under an explicit vault path; do not infer a default vault at runtime.                                                                                                                                          | Real CLI test verifies generated directories and index notes after `vault init`.                                                                       |
| Repo project registration                                       | true        | Git CLI for root/remote identity  | The wrapper owns mapping a repository to a central-vault project subtree and local pointer files. Git owns root and remote discovery.                   | Derive stable project IDs from a real Git remote; write complete `.agent-memory.toml`; create or replace the marked `AGENTS.md` memory section; fail loudly when no remote, explicit project ID, or well-formed AGENTS marker pair exists. | Real CLI tests initialize Git repos with a remote and prove project config, AGENTS guidance, existing AGENTS preservation, and vault subtree creation. |
| Note creation templates and frontmatter                         | true        | Filesystem plus IWE normalization | IWE templates can create notes, but the wrapper owns memory-specific frontmatter, scope, type, project ID, and provenance fields.                       | Enforce enumerated memory types/scopes plus OKF-compatible metadata through Pydantic; write plain Markdown notes under the configured scope; do not produce lifecycle state or action-sensitive authority fields.                          | Real CLI test creates project and global notes and asserts exact frontmatter fields, absent dropped fields, and body content.                          |
| Scoped retrieval                                                | true        | IWE `retrieve`                    | IWE owns graph-context retrieval; the wrapper owns loading project config and selecting the vault.                                                      | Run `iwe retrieve -k <key>` in the configured vault; propagate failure rather than substituting empty output.                                                                                                                              | Real CLI test retrieves a created note by key and proves the body returned is the real note content.                                                   |
| Graph consolidation                                             | true        | IWE `squash`                      | IWE owns inclusion traversal, inlining, and header adjustment; the wrapper owns config loading and an explicit public command.                          | Run `iwe squash <key> --depth <depth>` in the configured vault; require positive explicit depth; propagate failure rather than substituting output.                                                                                        | Real CLI test squashes a project index and proves dynamic project note content appears while unrelated global content does not.                        |
| Promotion from project to global memory                         | true        | IWE `rename` when moving keys     | The wrapper owns the promotion policy: project note becomes global, provenance is preserved, and a project pointer remains. IWE owns link-aware rename. | Move the note, update frontmatter to `scope: global`, add `origin_project_id`, and leave a project note pointing to the global key.                                                                                                        | Real CLI test promotes a project trap and proves old project note is a pointer while global note preserves original content and provenance.            |
| Doctor/setup verification                                       | true        | IWE/Git/rg subprocesses           | The wrapper owns user-facing setup diagnostics for its hard dependencies.                                                                               | Check configured vault, project config, and required binaries by invoking real commands; no optional dependency branches.                                                                                                                  | Real CLI test runs `doctor` after vault/project init and proves it reports the concrete project/vault contract.                                        |
