| TEST-007 | IOB-008          | CLI integration | `tests/test_cli_workflows.py` | Real temp vault path                                                                            | Run `python -m iwe2 vault init <vault>` through `uv run`                                                                    | Module execution creates the IWE config and global graph index links.                                                                                                                                                     | Package cannot be executed as a module or dispatches to a different command surface.                                                               | It proves package execution through a real subprocess and the same durable vault graph state.                                 |
| TEST-002 | IOB-002          | CLI integration | `tests/test_cli_workflows.py` | Real temp Git repos with remote, initialized vault, and absent, unmarked, or marked `AGENTS.md` | Run `iwe2 project init --vault <vault>` from the repo                                                                       | `.agent-memory.toml` has the derived project ID and vault; `AGENTS.md` names the current vault, project memory key, and scoped search command while preserving existing repo instructions; project index exists in vault. | Project config is written with wrong ID/root, project subtree is missing, AGENTS guidance is absent, or stale AGENTS memory guidance is preserved. | Git remote, existing Markdown, and file layout are real boundaries; expected ID is independently derived from the remote URL. |
| TEST-003 | IOB-003, IOB-004 | CLI integration | `tests/test_cli_workflows.py` | Initialized vault and project config                                                            | Create one project note and one global note, then run scoped searches.                                                      | Project search returns only project hit; global search returns only global hit; both returns both; generated frontmatter excludes lifecycle and action-sensitive fields.                                                  | Scope selection ignored, search rooted incorrectly, body search not wired to real files, or dropped frontmatter fields reintroduced.               | Uses distinct note bodies and the public CLI; arbitrary non-empty output fails.                                               |
| TEST-009 | IOB-004          | CLI integration | `tests/test_cli_workflows.py` | Initialized vault and project config                                                            | Create project and global notes whose titles fuzzy-match a query that is absent from note bodies.                           | Project title/key search returns only the project key; global title/key search returns only the global key; both returns both.                                                                                            | Scope anchors ignored, IWE `find` not wired, title/key matches dropped, or body-only search substituted for graph search.                          | The query is an acronym-style fuzzy match absent as a literal body string, so `rg` alone cannot satisfy the proof.            |
| TEST-010 | IOB-010          | CLI integration | `tests/test_cli_workflows.py` | Initialized vault and project config                                                            | Create project and global notes with a shared context token, then run `search-context` in project, global, and both scopes. | Probe JSON includes only project file hits for project scope, only global file hits for global scope, and both note files for both scope.                                                                                 | Probe not wired, wrong scope roots, only first root searched, JSON merge missing, or placeholder output returned.                                  | It runs real Probe through `npx` and checks dynamic temp-vault paths from notes created in the same test.                     |
| TEST-004 | IOB-005          | CLI integration | `tests/test_cli_workflows.py` | Note created through `iwe2 note`                                                                | Run `iwe2 retrieve <key>`                                                                                                   | Retrieved output includes the real note title and body.                                                                                                                                                                   | Retrieval bypasses IWE or returns placeholder text.                                                                                                | The note is created before retrieval in a temp vault; hard-coded output cannot satisfy dynamic content.                       |
| TEST-005 | IOB-006          | CLI integration | `tests/test_cli_workflows.py` | Project note created through `iwe2 note`                                                        | Run `iwe2 promote <project-key> --to global/traps`                                                                          | Global note preserves body/provenance; project key contains a pointer to the global key.                                                                                                                                  | Promotion deletes burden, loses content, or fails to leave project-local audit pointer.                                                            | It verifies both sides of the state transition on disk.                                                                       |
| TEST-006 | IOB-007          | CLI integration | `tests/test_cli_workflows.py` | Initialized vault and project config                                                            | Run `iwe2 doctor`                                                                                                           | Output names the exact vault path, derived project ID, and required tools.                                                                                                                                                | Doctor is a hollow facade or does not inspect configured state.                                                                                    | Dynamic temp paths and project IDs make hard-coded success text fail.                                                         |
| TEST-008 | IOB-009          | CLI integration | `tests/test_cli_workflows.py` | Project and global notes created through `iwe2 note`                                            | Run `iwe2 squash <project-index-key> --depth 3`                                                                             | Squashed output contains the dynamic project note title and body and excludes the dynamic global note body.                                                                                                               | Squash command missing, not delegated to IWE, rooted at the wrong graph key, or returning placeholder output.                                      | It invokes the public wrapper and verifies IWE output against dynamic note content from real Markdown files.                  |
