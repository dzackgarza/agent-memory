# Implementation Obligations

Source: [DESIGN-TRANSCRIPT.md](../../../DESIGN-TRANSCRIPT.md)

| ID | Requirement refs | Implementation component | Statement | Preconditions | Inputs | Expected outputs | State transitions | Error behavior | Delegated behavior | Local behavior | Evidence required |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| IOB-001 | central IWE workspace, single Git-versioned vault | `vault init` | Given an explicit vault path, initialize a complete central agent-memory vault layout backed by IWE. | Target path does not already contain an initialized IWE vault. | `vault init <vault>` | Root IWE config, `global/`, `projects/`, `inbox/`, `templates/`, `_meta/`, and root/global indexes exist. | New vault directory becomes the central memory store. | Existing initialized vault or failed IWE command fails the command. | `iwe init` creates the graph config. | Creates the memory-specific directory and index convention. | CLI integration test using a real temp directory. |
| IOB-002 | repo-local bootstrap pointer, project scoping | `project init` | Given a Git repository with a remote and an explicit vault path, register the repo as a project in the central vault and write a complete `.agent-memory.toml`. | Current directory is inside a Git repo; remote or explicit project ID is available; vault was initialized. | `project init --vault <vault>` | `.agent-memory.toml` contains `vault`, `project_id`, and `global_scopes`; project index exists under `projects/<project_id>/index.md`. | Repo becomes associated with the central vault. | Missing Git root, missing remote/project ID, or uninitialized vault fails. | Git CLI supplies root and remote. | Derives project ID, creates project subtree, writes local pointer config. | CLI integration test in a real Git repo with a remote. |
| IOB-003 | project/global note recording | `note` | Create a memory note in the selected scope with strict frontmatter and body content. | Vault/project config exists for project notes; vault exists for global notes. | `note --scope project --type decision --title ... --content ...` | Markdown note exists at the scoped location with exact required frontmatter and content. | Adds a durable note to project or global memory. | Invalid scope/type/status or missing project config fails through validation. | Filesystem stores Markdown; IWE can normalize after creation. | Maps scope/type/title to canonical note path and frontmatter. | CLI integration test reads the created Markdown file. |
| IOB-004 | scoped search | `search` | Search project, global, or both scopes over note bodies without leaving the configured vault roots. | Vault has project/global notes. | `search --scope project <query>` | Output contains only hits inside the selected scope with path and line numbers. | No state mutation. | Invalid scope or missing project config for project/both fails. | `rg` performs lexical body search. | Selects search roots and formats hits. | CLI integration test with distinct project/global note bodies. |
| IOB-005 | graph retrieval | `retrieve` | Retrieve a note with IWE context from the configured vault. | Vault is IWE-initialized; note key exists. | `retrieve <key>` | IWE retrieval output includes the target note content. | No state mutation. | IWE retrieval failure fails the wrapper. | IWE owns graph-context retrieval. | Resolves config and invokes IWE in the vault. | CLI integration test retrieves a note created by the wrapper. |
| IOB-006 | promotion | `promote` | Promote a project note to global memory, preserve provenance, and leave a project pointer. | Project config exists; source project note exists. | `promote <project-key> --to global/traps` | Global note contains original body, `scope: global`, and `origin_project_id`; project source key becomes a pointer to the global key. | Required memory changes from project-owned to global reusable memory. | Missing source note or invalid global destination fails. | IWE may update references through rename. | Applies memory promotion policy and provenance fields. | CLI integration test inspects both destination and pointer notes. |
| IOB-007 | setup diagnostics | `doctor` | Verify the configured memory environment through real dependencies and paths. | Project config exists. | `doctor` | Output identifies vault path, project ID, required binaries, and existing project/global roots. | No state mutation. | Missing binary/config/vault fails. | Git/IWE/rg commands prove required dependencies. | Reads config and checks the exact declared contract. | CLI integration test after successful project init. |
