| IOB-001 | central IWE workspace, single Git-versioned vault | `vault init`             | Given an explicit vault path, initialize a complete central agent-memory vault layout backed by Git and IWE.                                                                           | Target path does not already contain an initialized IWE vault.                                              | `vault init <vault>`                                                     | Git repository, staged root IWE config, `global/`, `projects/`, `inbox/`, `templates/`, `_meta/`, and root/global indexes exist.                                                                                   | New vault directory becomes the central memory store.                 | Existing initialized vault or failed Git/IWE command fails the command.                                           | Git initializes version control; `iwe init` creates the graph config.          | Creates the memory-specific directory and index convention, then stages vault changes.                                                            | CLI integration test using a real temp directory and real Git repository probe.                                              |
| IOB-002 | repo-local bootstrap pointer, project scoping     | `project init`           | Given a Git repository with a remote and an explicit vault path, register the repo as a project in the central vault and write complete `.agent-memory.toml` and `AGENTS.md` pointers. | Current directory is inside a Git repo; remote or explicit project ID is available; vault was initialized.  | `project init --vault <vault>`                                           | `.agent-memory.toml` contains `vault`, `project_id`, and `global_scopes`; `AGENTS.md` names the vault, project memory key, and scoped search command; project index exists under `projects/<project_id>/index.md`. | Repo becomes associated with the central vault.                       | Missing Git root, missing remote/project ID, uninitialized vault, or malformed existing IWE AGENTS section fails. | Git CLI supplies root and remote.                                              | Derives project ID, creates project subtree, writes local pointer config, and creates, appends, or replaces the marked AGENTS memory section.     | CLI integration tests in real Git repos with no `AGENTS.md`, existing unmarked `AGENTS.md`, and existing marked `AGENTS.md`. |
| IOB-003 | project/global note recording                     | `note`                   | Create a memory note in the selected scope with strict OKF-compatible frontmatter and body content.                                                                                    | Vault/project config exists for project notes; vault exists for global notes.                               | `note --scope project --type decision --title ... --content ...`         | Markdown note exists at the scoped location with exact required frontmatter, content, staged vault changes, and no lifecycle or action-sensitive fields.                                                           | Adds a durable staged note to project or global memory.               | Invalid scope/type or missing project config fails through validation.                                            | Filesystem stores Markdown; IWE can normalize after creation.                  | Maps scope/type/title to canonical note path and frontmatter without `status`, `authority`, `expires`, `safe_to_act`, or `requires_confirmation`. | CLI integration test reads the created Markdown file and Git status.                                                         |
| IOB-004 | scoped search                                     | `search`                 | Search project, global, or both scopes over note bodies and IWE title/key graph matches without leaving configured vault roots.                                                        | Vault has project/global notes.                                                                             | `search --scope project <query>`                                         | Body hits contain only selected-scope path/line matches; title/key hits contain only selected-scope memory keys.                                                                                                   | No state mutation.                                                    | Invalid scope or missing project config for project/both fails.                                                   | `rg` performs lexical body search; IWE `find` performs title/key graph search. | Selects search roots and graph anchors, then emits dependency output.                                                                             | CLI integration tests with distinct project/global note bodies and title/key-only fuzzy matches.                             |
| IOB-005 | graph retrieval                                   | `retrieve`               | Retrieve a note with IWE context from the configured vault.                                                                                                                            | Vault is IWE-initialized; note key exists.                                                                  | `retrieve <key>`                                                         | IWE retrieval output includes the target note content.                                                                                                                                                             | No state mutation.                                                    | IWE retrieval failure fails the wrapper.                                                                          | IWE owns graph-context retrieval.                                              | Resolves config and invokes IWE in the vault.                                                                                                     | CLI integration test retrieves a note created by the wrapper.                                                                |
| IOB-006 | promotion                                         | `promote`                | Promote a project note to global memory, preserve provenance, and leave a project pointer.                                                                                             | Project config exists; source project note exists.                                                          | `promote <project-key> --to global/traps`                                | Global note contains original body, `scope: global`, and `origin_project_id`; project source key becomes a pointer to the global key.                                                                              | Required memory changes from project-owned to global reusable memory. | Missing source note or invalid global destination fails.                                                          | IWE may update references through rename.                                      | Applies memory promotion policy and provenance fields.                                                                                            | CLI integration test inspects both destination and pointer notes.                                                            |
